# webportal-leyu-guide

本仓库用于归档和发布多个独立 HTML 页面。页面内容涵盖不同主题，以静态页面形式存放，便于浏览与索引。

## 项目简介

此仓库主要用于存放与网页入口、导航或信息展示相关的 HTML 文件。每个页面均为独立文件，不依赖特定后端环境，可直接通过浏览器打开或在 GitHub Pages 上发布。

## 目录说明

```
/
├── index.html       # 仓库首页或默认入口页面
├── pages/           # 存放各类独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── assets/          # 共用资源文件（如图片、样式、脚本）
    ├── css/
    ├── js/
    └── img/
```

- `index.html`：仓库默认展示页面，可作为页面总览或导航。
- `pages/`：存放各独立 HTML 页面，每个文件一个主题。
- `assets/`：存放被多个页面引用的共用资源，便于统一管理。

## 页面归档说明

- 所有页面均为独立 HTML 文件，不依赖外部数据库或服务器端脚本。
- 页面之间无强制关联，可单独访问或修改。
- 建议在 `pages/` 目录下按主题或用途对文件进行归类，并在 `index.html` 中提供链接索引。

## 维护说明

- 新增页面时，请将 HTML 文件放入 `pages/` 目录，并保持文件命名清晰。
- 若页面引用 `assets/` 中的资源，请使用相对路径。
- 修改现有页面时，请注意保持页面结构与风格的一致性。
- 本仓库不涉及具体服务或推广内容，仅作为静态页面归档使用。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开任意 HTML 文件即可查看。
3. 若启用 GitHub Pages，可通过仓库设置将 `main` 分支发布为静态站点。
